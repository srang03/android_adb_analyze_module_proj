## ✅ 논문 작성에 문제가 없는 이유

### 1. **재부팅 휘발성은 예상 가능한 현상**

재부팅 시 로그가 휘발되는 것은 Android 시스템의 **정상적인 동작**입니다:

- `media_camera.log`: **메모리 기반 로그** → 재부팅 시 완전 삭제
- `audio.log`: 메모리 기반 → 재부팅 시 완전 삭제
- `vibrator_manager.log`: 메모리 기반 → 재부팅 시 완전 삭제
- `activity.log`: 메모리 기반 → 재부팅 시 완전 삭제

**결론**: 재부팅 후 0% 탐지율은 **시스템의 한계**이지 **도구의 결함이 아님**

### 2. **24시간 휘발성과의 차별화된 발견**

| 휘발성 유형 | media_camera.log | 탐지율 | 의미 |
|-----------|------------------|--------|------|
| **24시간 휘발성** (Sample 2, 3, 5) | ✅ 보존 (44개) | **100%** | 시간 경과에도 견고함 |
| **재부팅 휘발성** (Sample 1) | ❌ 완전 휘발 (0개) | **0%** | 재부팅 시 한계 존재 |

**논문 기여**:
- ✅ **24시간 휘발성 환경에서 100% 탐지율** → 도구의 실용성 입증
- ✅ **재부팅 휘발성 환경에서 0% 탐지율** → 도구의 한계 명확히 제시
- ✅ **두 휘발성 메커니즘의 차이 규명** → 학술적 가치 증대

### 3. **실무 포렌식 환경에서의 시사점**

재부팅 후 0% 탐지율은 실무에서 중요한 의미를 가집니다:

| 시나리오 | 탐지 가능성 | 포렌식 대응 방안 |
|---------|-----------|---------------|
| 휴대폰 압수 후 바로 분석 | ✅ 높음 | 즉시 로그 추출 |
| 24시간 이내 분석 | ✅ 높음 (100%) | 정상적인 분석 절차 |
| 재부팅 후 분석 | ❌ 낮음 (0%) | ⚠️ 재부팅 방지 조치 필요 |

**논문 기여**:
- ✅ 실무자에게 **"재부팅 전 로그 추출"의 중요성** 강조
- ✅ 포렌식 절차 개선을 위한 **구체적인 가이드라인** 제시
